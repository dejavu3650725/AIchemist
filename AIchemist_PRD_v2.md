# Product Requirements Document (PRD)
## 프로젝트명: AIchemist (학습용 웹앱 포털)

### 1. 프로젝트 개요 (Overview)
'AIchemist'는 개발자가 자체 제작한 다양한 학습용 웹앱들을 한곳에 모아 관리하고, 사용자가 쉽고 빠르게 접근할 수 있도록 돕는 웹 포털입니다. 스마트폰이나 태블릿의 '앱 런처(App Launcher)'와 유사한 직관적인 UI를 제공하여, 사용자가 원하는 학습 도구를 즉시 찾고 실행할 수 있는 통합 환경을 구축합니다.

### 2. 핵심 목표 (Objectives)
- **중앙 집중화**: 흩어져 있는 개별 학습용 웹앱들을 단일 진입점(Portal)에서 통합 제공.
- **직관적 접근성**: 텍스트 위주의 복잡한 화면을 배제하고, 아이콘 기반의 앱 런처 방식으로 최소한의 클릭으로 앱 실행.
- **유연한 확장성**: 추후 새로운 학습 앱이 개발/추가될 때 쉽게 포털 메타데이터에 등록하고 카테고리화할 수 있는 유연한 구조 마련.

### 3. 사용자 스토리 (User Stories)
- 사용자는 포털 메인에 접속하여 사용할 수 있는 **전체 학습 앱 목록**을 앱 아이콘 형태의 그리드로 한눈에 볼 수 있다.
- 사용자는 화면 좌측이나 상단의 메뉴를 통해 '어학', '수학', '코딩' 등 **카테고리별**로 원하는 앱만 필터링해서 볼 수 있다.
- 사용자는 상단의 검색창을 통해 앱 이름이나 기능 키워드로 원하는 앱을 빠르게 검색할 수 있다.
- 사용자는 앱 아이콘을 클릭하여 해당 학습 앱을 새 탭(또는 팝업)에서 즉시 실행할 수 있다.
- **[추가] 관리자는 별도의 복잡한 로그인 없이 '앱 추가' 버튼을 누르고 '관리자 코드'를 인증하여 새로운 앱을 포털에 직접 등록할 수 있다.**

### 4. 화면 및 UI/UX 요구사항 (UI/UX Architecture)

#### 4.1. 메인 화면 (App Launcher View)
- **그리드 레이아웃**: 데스크탑 화면 기준 4~6열, 모바일 기준 2~3열의 반응형 아이콘 그리드 구성.
- **앱 카드/아이콘 디자인**: 
  - 시인성이 높은 앱 고유 아이콘 (또는 통일성 있는 일러스트)
  - 아이콘 하단에 간결한 앱 타이틀 배치
  - 마우스 오버(Hover) 시 앱에 대한 1~2줄의 짧은 설명과 태그 툴팁 노출

#### 4.2. 네비게이션 (Navigation & Filtering)
- **사이드바 (또는 상단 탭)**:
  - `전체 앱 (All Apps)` : 디폴트 화면
  - `즐겨찾기 (Favorites)` / `최근 실행 (Recent)`
  - `카테고리 분류`: ex) 어학, 프로그래밍, 데이터 분석, 생산성 도구 등
- **통합 검색 (Search Bar)**: 화면 상단 중앙에 위치하며, 입력 즉시 아래 런처 그리드가 필터링되는 실시간 검색 지원.

#### 4.3. 앱 실행 방식 (App Execution)
- 기본적으로 **새 탭(_blank)**으로 개별 웹앱 URL을 띄워 넓은 화면에서 학습을 진행하도록 함.
- (선택 사항) 계산기나 단어장 같은 가벼운 앱은 포털 화면 이탈 없이 화면 중앙의 **모달(Iframe)** 형태로 실행.

#### 4.4. 관리자 모드: 간편 앱 등록 (Admin Registration)
- **접근 및 인증**: 화면 우측 하단이나 헤더 영역에 눈에 띄지 않게 배치된 '+' (또는 설정) 버튼을 클릭 후, 사전에 설정된 **관리자 인증 코드(Admin Code/Password)**를 입력하여 권한 획득.
- **앱 등록 폼 (Form View)**: 인증 완료 시 나타나는 모달 창.
  - `카테고리 선택`: 미리 정의된 카테고리(수학, 어학, 코딩 등) 중 드롭다운으로 선택.
  - `제목 입력 (Title)`: 사용자에게 표시될 앱의 이름.
  - `링크 입력 (URL)`: 해당 학습 앱으로 연결될 실제 웹주소.
  - `썸네일 업로드 (Thumbnail)`: 앱 런처에 보여질 이미지/아이콘 파일 업로드.
- **실시간 반영**: 등록하기(Submit) 버튼을 누르면 DB(또는 설정된 JSON 파일)에 데이터가 추가되며, 즉시 메인 화면에 썸네일과 함께 새로운 앱이 나타남.

### 5. 데이터 명세 (Data Structure)
포털에서 앱 목록을 렌더링하기 위한 JSON 메타데이터 구조 (예시):
```json
{
  "appId": "lang-001",
  "title": "영단어 연성진",
  "description": "매일 50개의 영단어를 퀴즈 형태로 학습합니다.",
  "icon": "https://aichemist.dev/uploads/thumb_vocab.jpg", 
  "url": "https://aichemist.dev/vocab",
  "category": "Language",
  "tags": ["영어", "단어장", "토익"]
}
```

### 6. 릴리즈 단계별 계획 (Milestones)

#### Phase 1: MVP (Minimum 기획)
- 앱 런처 메인 UI, 카테고리 필터링, 실시간 검색 기능 완성.
- **관리자 코드 인증 기반 간편 앱 등록 폼(카테고리, 제목, URL, 썸네일) 및 실시간 렌더링 기능 구현.**
- 외부 링크(새 탭) 연결.

#### Phase 2: 개인화 및 편의성 강화
- LocalStorage를 활용한 '최근 실행한 앱' 및 '즐겨찾기' 기능 추가.
- 다크모드 / 라이트모드 테마 토글 기능 적용.

#### Phase 3: 게이미피케이션 및 대시보드 (Long-term)
- 사용자 로그인 도입 및 개별 학습 앱과의 계정 연동 (SSO).
- 각 앱의 학습 진도율, 획득한 경험치(EXP) 등을 포털 메인 상단 위젯으로 요약 표시.
